# Cloud Armor: Traffic Blocklisting (WAF at the Edge)

## Goal
Block malicious/abusive traffic at the edge before it reaches the VPC / backend services.

## What you will prove (evidence)
- A Cloud Armor security policy exists and is attached to the backend service.
- A deny rule triggers (blocked request) and is visible in logs.
- Baseline traffic still works (allowed request).
